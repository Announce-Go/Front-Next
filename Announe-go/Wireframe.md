## 1. 공통 화면 상세

### 1.1 로그인 화면

- **경로**: `/login`

#### 레이아웃

```
┌─────────────────────────────────────┐
│         알리다고 로고/제목           │
│                                     │
│  ┌───────────────────────────┐     │
│  │  아이디                   │     │
│  └───────────────────────────┘     │
│  ┌───────────────────────────┐     │
│  │  비밀번호                 │     │
│  └───────────────────────────┘     │
│                                     │
│  □ 로그인 상태 유지                 │
│                                     │
│      [로그인]                       │
│                                     │
│  ─────────────────────────────      │
│                                     │
│        회원가입                     │
└─────────────────────────────────────┘
```

#### 데이터 처리

- 로그인 API 호출
  - 요청: 로그인ID, 비밀번호, 로그인상태유지여부
  - 응답: 인증토큰, 사용자타입(admin/agency/hospital), 승인상태

#### 화면 전환

- 로그인 성공:
  - 관리자 → `/admin/dashboard`
  - 광고업체 → `/agency/dashboard`
  - 병원 → `/hospital/dashboard`
- 승인 대기: "승인 대기 중" 메시지 표시
- 회원가입 링크 → `/register`

#### 특이사항

- 좌측 네비게이션은 모든 관리자 화면에서 공통으로 표시됨
- 현재 선택된 메뉴는 하이라이트 표시
- 승인 대기 목록은 최근 5건만 표시, 더 많은 내용은 [전체 보기]로 확인

- 반려 시 사유는 필수 입력

#### 데이터 처리

- 기사 목록 조회 API 호출
  - 요청: 연월(YYYY-MM)
  - 응답: 해당월의 기사목록(날짜별 그룹화)
- 특정 날짜 기사 조회 API 호출
  - 요청: 날짜(YYYY-MM-DD)
  - 응답: 해당 날짜의 기사목록
- 기사 등록 API 호출
  - 요청: 날짜, 내용, 광고주ID
  - 응답: 처리결과
- 기사 수정 API 호출
  - 요청: 기사ID, 내용, 광고주ID
  - 응답: 처리결과
- 기사 삭제 API 호출
  - 요청: 기사ID
  - 응답: 처리결과

#### 화면 전환

- [◀] [▶] 클릭 → 이전/다음 달로 이동, 캘린더 갱신
- 캘린더의 날짜 클릭 → 하단에 해당 날짜의 기사 목록 표시
- [+ 추가] 클릭 → 하단에 기사 등록 폼 표시
- [수정] 클릭 → 하단에 기사 수정 폼 표시 (해당 기사 내용 미리 채움)
- [삭제] 클릭 → 확인 모달 → 삭제 처리 → 목록 갱신
- [취소] 클릭 → 하단 폼 닫기
- [저장] 클릭 → 등록/수정 처리 → 폼 닫기 → 목록 갱신

#### 특이사항

- 캘린더에서 기사가 있는 날짜는 건수 표시 (예: 16•2 = 16일에 2건)
- 관리자가 등록/수정/삭제 가능 (별도 페이지 이동 없이 동일 화면에서 처리)
- 업체/광고주는 조회만 가능
- 날짜 클릭 시에만 하단에 해당 날짜의 기사 목록 표시
- [+ 추가] 또는 [수정] 클릭 시 하단에 폼 표시 (목록 아래에 인라인으로 표시)
- 날짜 필드는 클릭한 날짜로 자동 설정
- 광고주 선택은 드롭다운 (승인된 광고주 목록)
